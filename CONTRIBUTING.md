# Contributing to OpsQuest

OpsQuest teaches support judgment through realistic practice. Contributions to
scenarios, coaching feedback, accessibility, documentation, and the app itself
are welcome—especially from people who have worked a real support queue.

## Content principles

- Teach the reasoning, not a magic command.
- Reward identity verification, communication, documentation, and escalation
  judgment alongside technical correctness.
- Use fictional companies, people, domains, and incidents. Never adapt a real
  customer ticket closely enough to identify it.
- Keep feedback direct, specific, and kind.

## Run locally

```bash
npm install
npm start
```

Use `npm run web` for the quickest UI review. For app changes, include the
platforms you tested. For scenario changes, walk every choice path and include
screenshots of the scenario and debrief.

Small fixes may go directly to a pull request. Please open an issue before a
new scenario system, scoring model, or major navigation change.
