“The technically correct answer delivered badly is still a failed ticket.”

Support is technical and human at the same time.

OpsQuest scores both sides of the job. It rewards good troubleshooting, but it also trains the soft edges that keep clients steady: plain-English updates, security confidence, ownership, and notes the next tech can trust.

Local-first, server-optional

Self-host the content server.

OpsQuest works offline with no account required. Teams can optionally self-host the content server to publish new scenarios, using the GHCR image ghcr.io/spillers-technology/opsquest-server. It includes a web scenario editor for managing content.

Author in plain JSON

Scenarios use a documented JSON format with a validator, so teams can write and check new content without changing app code.
